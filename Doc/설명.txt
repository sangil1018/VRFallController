안전체험 VR 시뮬레이터 매니저

[  ] vr기기 pico4 3대를 제어하고 1첫째 기기의 화면 미러링을 함.
[  ] 시뮬레이터와 통신을해서 직접 기기 제어를 함.
[  ] 또는 첫번째 연결된 피코4 에서 신호를 받아 시뮬레이터를 제어함. 두번째를 선호하지만 두방식 모두 필요함.


안전체험 콘텐츠는 기존 유니티 콘텐츠가 있음.
[  ] 시뮬레이터로 콘텐츠가 시작할때 시작신호 엘리베이터가 올라갈때 신호, 멈춤 신호 또는 올라가서 정지, 추락할때 신호 전달. 필요시 초기화 전달해야하나 현재 시물레이터 프로토콜이 없어서 구성만해둠.


PC 제어 앱 구성
0. 테스트 모드
(개발단계에서만 TEST 심볼로 제어)

1. 시뮬레이터 연결상태 창 (대기/연결/끊김), 스캔

2. 체험 컨트롤
[  ] 체험 시작 > 피코 디바이스에 play 신호 전달. 
시뮬레이터 포트열기(프로토콜에 따라서 수정가능)
[  ] 시뮬레이터 제어 자동/수동 선택
자동일땐 하위 메뉴 숨김. 1번 피코에서 신호 전달 받아 시뮬레이터 제어.
수동일땐 하위 메뉴 보임. 상승 타이밍, 상승 시간(초)/ 추락 타이밍, 추락 시간 입력 및 신호 전달.
[  ] 체험 멈춤/계속, 종료 > pause/resume 신호, stop 신호 전달 컨텐츠 멈춤, 종료(초기화)

3.adb 컨트롤
[  ] 피코 디바이스 검색 및 목록 등록 자동, 수동 스캔
[  ] 등록한 디바이스에 설치한 apk 전체 실행 (오차 최소화), 디바이스 개별 선택해서 실행가능
[  ] 콘텐츠 종료 > 전체, 개별선택 종료가능
[  ] 디바이스 재부팅  > 전체, 개별 가능
[  ] apk선택해서 목록에 보이는 기기들에 설치, 삭제. 전체, 선택해서 개별적용가능
[  ] 작동방식 버튼을 누르면 등록된 전체 디바이스에 작용 리스트에서 선택하면 선택한 디바이스들만 적용.

4. log window > 색상으로 중요도 구분.


유니티 클라이언트 구성
[  ] 시작, 멈춤/다시시작, 종료 신호 받아 timeline 재생에 적용.
[  ] 재생중 엘리베이터 업, 추락 시 신호를 전달 > 제어앱으로 전달 또는 시물레이터에 직접 제어신호 전달
[  ] 재생 종료 후 timeline 초기화.

테스트모드 구성.
[  ] 가상 피코 (안드로이드) 에뮬레이터를 만들어서 ip 192.168.1.101, 102, 103 할당하고, adb 명령이 전달되면 적절한 리턴값을 전달해서 실제 피코 디바이처럼 속여 테스트가 가능하도록 구성
[  ] 가상 시뮬레이터 에뮬레이터를 만들어서 제어앱 또는 피코 디바이스에 설치된 유니티로 부터 신호를 전달받아 적절한 리턴값을 전달
[  ] 피코 에뮬레이터, 시뮬레이터 에뮬레이터로 속여 pc 제어앱 구동 테스트
